3.Bean的常用配置

id 和 name

  • 一般情况下，装配一个Bean时，通过指定一个id属性作为bean的名称
  • id属性在ioc容器里是唯一的
  • 如果bean的名称含有特殊字符，就需要使用name属性了(除此之外，name和id没有区别)

class

  • class用于设置一个类的完全路径名称，主要作用是IOC容器生成类的实例。

scope(4种)(Bean的作用范围)

  • singleton 实例为单例(默认)
  • prototype 每次new一个实例都是一个新的,多例
  • request
  • session

生命周期(指定函数，在bean初始化/销毁的时候做事情):

  • init-method
  • detory-method (只有scope为singleton的时候才有用)

5.Spring属性注入

构造函数注入

首先你得有个构造函数

然后用constructor-arg来注入

setter方法注入(更喜欢)

首先你得有个setter函数

然后用property来注入

如果在bean里有一个bean接口的话，就用 ref=”#另一个bean的id” 来注入

p空间注入

可以在IOC后面加

p:(属性)=“”

p:(属性-ref)=””

SPEL注入

这个和p注入一样，就是为了简化注入操作的。

可以使用另外一个类的属性或方法来注入。
